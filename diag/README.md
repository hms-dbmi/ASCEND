# ASCEND-Diag

ASCEND-Diag estimates diagnostic probabilities for observed *de novo* variants using ASCEND-derived reference cohort summaries, mutation-rate annotations, variant functional class, and missense pathogenicity scores.

ASCEND-Diag is distinct from the main ASCEND association-testing workflow. The main ASCEND pipeline tests whether genes are enriched for *de novo* variation. ASCEND-Diag instead estimates the probability that a specific observed variant contributes to the phenotype of an individual proband, under the assumptions of the ASCEND-Diag model.

## Contents of this directory

```text
ASCEND_Diag.py
gene_list_file.txt

NDD_ref_var_info.txt
NDD_ref_sum_stats.txt

ASD_ref_var_info.txt
ASD_ref_sum_stats.txt

CHD_ref_var_info.txt
CHD_ref_sum_stats.txt

EE_ref_var_info.txt
EE_ref_sum_stats.txt
```

The `*_ref_var_info.txt` and `*_ref_sum_stats.txt` files are intermediate outputs generated by the main ASCEND algorithm for the NDD, ASD, CHD, and EE reference cohorts. They are used here as precomputed diagnostic reference cohorts.

## Script

The main script is:

```bash
ASCEND_Diag.py
```

## Required inputs

ASCEND-Diag requires four main inputs.

### 1. Gene list

A text file containing the genes to evaluate, with one Ensembl gene ID per line.

The provided file is:

```text
gene_list_file.txt
```

Lines beginning with `#` are ignored.

### 2. Reference cohort variant-info file

A tab-separated file containing annotated variants observed in the selected reference disease cohort. These files are generated as intermediate ASCEND outputs and are provided for four reference cohorts:

```text
NDD_ref_var_info.txt
ASD_ref_var_info.txt
CHD_ref_var_info.txt
EE_ref_var_info.txt
```

Required columns include:

```text
ensembl_gene_id
SAMPLE_RECURRENCE
Synonymous-flag
LOFTEE-flag
AlphaMissense_MT
REVEL_MT
PrimateAI-3D_MT
```

### 3. Reference cohort per-gene summary-statistics file

A tab-separated file containing per-gene summary statistics and mutation-rate expectations for the selected reference disease cohort. These files are also generated as intermediate ASCEND outputs and are provided for four reference cohorts:

```text
NDD_ref_sum_stats.txt
ASD_ref_sum_stats.txt
CHD_ref_sum_stats.txt
EE_ref_sum_stats.txt
```

Required columns include:

```text
ENS_ID
syn_varN
Lof_varN
Lof_lambda
AM_lambda
REVEL_lambda
PAI_lambda
```

### 4. Processed target-variant file

A tab-separated file containing the variants for which diagnostic probabilities should be computed. This file may be plain text or gzip-compressed.

Required columns include:

```text
ensembl_gene_id
Synonymous-flag
LOFTEE-flag
AlphaMissense_MT
REVEL_MT
PrimateAI-3D_MT
```

Additional columns are preserved in the output.

## Basic usage

Example using the NDD reference cohort:

```bash
python ASCEND_Diag.py \
  --gene-list-file gene_list_file.txt \
  --ref-var-info-file NDD_ref_var_info.txt \
  --ref-sum-stats-file NDD_ref_sum_stats.txt \
  --processed-vars-file target_variants.tsv.gz \
  --output-file target_variants_NDD_diag.tsv.gz \
  --colname NDD \
  --P_D 0.01 \
  --pnull 0.25 \
  --add-one-to-varN \
  --verbose
```

To use another reference cohort, replace both reference files with the matching pair. For example, for ASD:

```bash
python ASCEND_Diag.py \
  --gene-list-file gene_list_file.txt \
  --ref-var-info-file ASD_ref_var_info.txt \
  --ref-sum-stats-file ASD_ref_sum_stats.txt \
  --processed-vars-file target_variants.tsv.gz \
  --output-file target_variants_ASD_diag.tsv.gz \
  --colname ASD \
  --P_D 0.01 \
  --pnull 0.25 \
  --add-one-to-varN
```

## Command-line options

```text
--gene-list-file
    File containing the gene whitelist, one Ensembl gene ID per line.
    Default: gene_list_file.txt

--ref-var-info-file
    Reference cohort variant-info file.

--ref-sum-stats-file
    Reference cohort per-gene summary-statistics file.

--processed-vars-file
    Variants for which diagnostic probabilities should be calculated.
    May be plain text or gzip-compressed.

--output-file
    Output file. If the filename ends in .gz, output is gzip-compressed.

--colname
    Prefix for added diagnostic-probability columns.
    Default: DiagProb

--P_D
    Assumed prevalence of the umbrella phenotype.
    Default: 0.01

--pnull
    Assumed probability of LoF misannotation.
    Default: 0.25

--add-one-to-varN
    Add one to the observed variant count in enrichment calculations.
    This is the intended mode when evaluating an incoming proband whose
    variant is not already included in the reference cohort.

--verbose
    Print progress messages.
```

## Output

ASCEND-Diag writes the input target-variant table with additional diagnostic-probability columns appended.

For each missense score, four diagnostic-probability estimates are reported:

```text
<colname>_PP_AlphaMissense_MT_1
<colname>_PP_AlphaMissense_MT_2
<colname>_PP_AlphaMissense_MT_3
<colname>_PP_AlphaMissense_MT_4

<colname>_PP_REVEL_MT_1
<colname>_PP_REVEL_MT_2
<colname>_PP_REVEL_MT_3
<colname>_PP_REVEL_MT_4

<colname>_PP_PrimateAI-3D_MT_1
<colname>_PP_PrimateAI-3D_MT_2
<colname>_PP_PrimateAI-3D_MT_3
<colname>_PP_PrimateAI-3D_MT_4
```

For LoF variants, the enrichment-based LoF diagnostic probability is copied across these output columns. For synonymous variants and variants that are neither LoF nor scored missense, diagnostic-probability columns are set to `NA`.

The four missense estimates correspond to different ways of estimating diagnostic probability:

```text
1. Gene-level missense enrichment.
2. Genome-wide score-based pathogenicity model.
3. Gene-specific score-based pathogenicity model.
4. Combined score and enrichment-based model.
```

## Reference cohorts

The provided reference files correspond to:

```text
NDD  neurodevelopmental disorders
ASD  autism spectrum disorder
CHD  congenital heart disease / congenital diaphragmatic hernia
EE   encephalopathic epilepsy
```

For each phenotype, the `*_ref_var_info.txt` and `*_ref_sum_stats.txt` files should be used as a matched pair. For example, use `NDD_ref_var_info.txt` with `NDD_ref_sum_stats.txt`.

Users may also generate their own reference cohort files by running the main ASCEND workflow on another disease cohort and retaining the corresponding variant-info and per-gene summary-statistics intermediate outputs.

## Relationship to main ASCEND data

ASCEND-Diag uses the ASCEND mutation-rate and annotation framework. In particular, the target variants should be processed using the same annotation conventions as the main ASCEND pipeline, including mutation-rate-weighted missense score columns such as:

```text
AlphaMissense_MT
REVEL_MT
PrimateAI-3D_MT
```

The large core ASCEND annotation file,

```text
muttargs5_noOL_RQC_format.txt.gz
```

is stored in the main ASCEND `data/` directory rather than this `diag/` directory.

## Notes and limitations

ASCEND-Diag assumes that the incoming proband and the selected reference disease cohort are ascertained under a comparable umbrella phenotype. Diagnostic probabilities therefore depend on the choice of reference cohort, the assumed phenotype prevalence `P_D`, the variant annotation model, and the missense pathogenicity score calibration.

ASCEND-Diag is research-use software. Its output should not be used as the sole basis for clinical diagnosis or medical decision-making.
